## 🧪 AI Testing Labs

Hands-on labs built to answer one question: *how do you actually test AI systems?* Working code, not just concepts.

| Lab | What it covers |
|---|---|
| [behavioral-invariant-testing](https://github.com/martin-kratmuller/behavioral-invariant-testing) | Rules that must never break, regardless of model changes |
| [behavioral-regression-ai](https://github.com/martin-kratmuller/behavioral-regression-ai) | Scenario-based regression with tolerance bands instead of golden outputs |
| [ml-integration-boundary-testing](https://github.com/martin-kratmuller/ml-integration-boundary-testing) | Contract tests, messy real-world inputs, async invariants |
| [adversarial-ml-testing](https://github.com/martin-kratmuller/adversarial-ml-testing) | Gibberish, prompt injection, extreme inputs — failing safely |
| [ml-data-quality-testing](https://github.com/martin-kratmuller/ml-data-quality-testing) | Data validation before training: schema, balance, duplicates, drift |
| [model-version-comparison](https://github.com/martin-kratmuller/model-version-comparison) | V1 vs V2 side-by-side: catching regressions behind better accuracy |
| [llm-prompt-testing](https://github.com/martin-kratmuller/llm-prompt-testing) | 5 prompt formats, cross-prompt consistency, injection resistance |

All labs: Python + pytest.
